## STATEMENT OF WORK

Client: [AMERICAN BEDDING LEGAL ENTITY NAME -- TBD]

### 1. Executive Summary

American Bedding is a Vesco portfolio company manufacturing mattresses for camps, educational institutions, and government agencies. With NetSuite launched January 1, 2026 and HubSpot in place for CRM, the infrastructure is ready -- but the quoting process remains fragmented across four systems (NetSuite, Excel, Kuebix TMS, and HubSpot).

Sayer is implementing HubSpot CPQ as the single quoting interface, building automated integrations with NetSuite and Kuebix, and replacing manual Excel weight calculators with a dynamic calculation engine. The engagement is structured in two phases:

- **Phase 1 (V1):** HubSpot-Native CPQ -- Get sales reps quoting in HubSpot with automated freight, product catalog sync, and quote-to-order integration. Delivered over 12 weeks.
- **Phase 2 (V2):** Production Platform -- Migrate integration logic to a dedicated cloud API for unlimited performance, full test coverage, and long-term maintainability. Delivered over 6 weeks following Phase 1.

When complete, creating a quote -- from product selection through freight calculation to customer delivery -- happens in one system.

### 2. Objectives

1. **Single-Interface Quoting** -- Sales reps create, edit, and send quotes entirely within HubSpot. No system-jumping for standard inside sales quotes.
2. **Automated Freight Calculation** -- Kuebix API integration calculates multi-carrier LTL shipping rates automatically with cheapest rate auto-selected and manual override option.
3. **Dynamic Weight Engine** -- Replaces five Excel spreadsheets with an automated calculation engine that determines product weight and volume from specifications.
4. **Product Catalog in HubSpot** -- All 700 SKUs synced from NetSuite with pricing, specifications, and custom attributes. Ongoing automated sync.
5. **Quote-to-Order Automation** -- Accepted quotes automatically create Estimates in NetSuite. Approved estimates convert to Sales Orders.
6. **Branded Quote Templates** -- Professional quote PDFs with line items, discounts, tax, freight, terms, and customer acceptance.
7. **Credit Hold Visibility** -- Customer credit status from NetSuite surfaces in HubSpot before quote creation.
8. **Volume Discount and Tax Automation** -- 10% volume discount logic and state-based tax calculation built into the quoting workflow.
9. **Multi-Warehouse Freight Routing** -- Freight quotes automatically use the correct origin warehouse based on product and order characteristics.
10. **Full Quote Visibility for Leadership** -- Quote activity, versions, edits, approvals, and cancellations visible in HubSpot dashboards.

### 3. Scope of Work

#### Phase 1 (V1): HubSpot-Native CPQ -- Weeks 1-12

**Workstream 1: CPQ Architecture and Configuration**

Commerce Hub CPQ module activation, custom deal/quote/line item properties, quote approval workflows, quote numbering, and payment terms configuration.

Assumptions:
- Requires HubSpot Sales Hub Professional + Commerce Hub Professional (existing)
- Requires Operations Hub Professional for custom code actions (~$800/month if not already licensed)
- Includes up to 30 custom properties across deals, quotes, and line items

**Workstream 2: Product Catalog Sync (NetSuite to HubSpot)**

All 700 SKUs extracted from NetSuite via SuiteQL and loaded into HubSpot with custom properties: weight components, freight class, dimensions, cover type, foam series, and construction type. Automated sync keeps HubSpot pricing current with NetSuite changes.

Assumptions:
- Flat product catalog (one SKU per product combination, no variant system)
- Up to 700 products in HubSpot product library
- Client provides NetSuite admin access or creates OAuth 2.0 M2M integration record

**Workstream 3: NetSuite Integration**

OAuth 2.0 M2M authentication. Customer data sync including credit status and payment terms. Price level synchronization. Bi-directional data flow between HubSpot and NetSuite.

Assumptions:
- NetSuite REST API with OAuth 2.0 M2M authentication
- Client provides or authorizes creation of integration credentials
- TPI (NetSuite implementer) has not locked down the environment in a way that blocks API access

**Workstream 4: Kuebix Freight Integration**

HubSpot custom code connects quotes to Kuebix's multi-carrier LTL rate shopping API. For each quote, the system collects product weights and dimensions, determines the correct shipping origin, requests rates from all configured carriers, and writes the best rate as a freight line item. Manual override available for truckload or special situations.

Assumptions:
- Client provides Kuebix API credentials (username, API key, Client ID)
- Kuebix quickRate API endpoint for LTL freight
- Rate limits sufficient for 65+ quotes/week volume

**Workstream 5: Dynamic Weight Calculation Engine**

Replaces five Excel spreadsheets with a modular calculation engine. Each product line gets its own calculation module: Dorm Mattresses (innerspring, foam, batting, covers), Camp Mattresses (foam cores with fire barriers), Dura-Last (poly fiber fill), SoFlux and Vinyl (covers only). Shared utilities handle common math (fabric area, cube volume, cover material weights, packaging). Calculates shipping weight and cube volume from product specifications for both standard and custom configurations.

Assumptions:
- Five product-line calculation modules built from validated Excel files (received April 4, 2026)
- Dorm Mattress is primary complexity driver (~180 calculation paths)
- "Special Case Check with Delbert" business logic to be clarified at kickoff
- HubSpot custom code action execution (20-second limit applies in Phase 1)

**Workstream 6: Quote Template Design**

Custom HubSpot quote template matching current NetSuite format. Line item table with quantity, item number, description, unit price, and amount. Subtotal, volume discount, state-based tax, shipping, and total. Damaged freight policy terms. Delivery instructions. Customer acceptance flow.

Assumptions:
- One branded quote template
- Sarah-Beth Knight leads artwork/branding design decisions
- Customer acceptance via clickwrap or e-signature within HubSpot

**Workstream 7: Discount, Tax, and Payment Terms Automation**

Volume discount automation (10% threshold), state-based tax calculation, payment terms workflow (prepay, net 10, net 30, net 60), and credit hold integration from NetSuite accounts receivable.

Assumptions:
- Standard discount and tax rules as documented in discovery
- Credit hold data sourced from NetSuite AR

**Workstream 8: Quote-to-Order Bridge (HubSpot to NetSuite)**

When a customer accepts a quote, an Estimate is created in NetSuite. On approval, the Estimate converts to a Sales Order, triggering production work orders and shipping preparation. No manual re-entry between systems.

Assumptions:
- NetSuite Estimate and Sales Order creation via REST API
- Standard quote-to-order flow (no custom approval chains beyond what was discussed)

**Workstream 9: Testing, Training, and Go-Live**

End-to-end integration testing across all three systems. Quote accuracy validated against five NetSuite example quotes. Weight calculations validated against existing Excel spreadsheets. Freight calculations validated against manual Kuebix lookups. Two UAT sessions with Caleb and Don. Sales team training (Caleb, Don) and admin training (Sarah-Beth, Patrick). Go-live cutover with 2-week hypercare period.

Assumptions:
- Caleb and Don available for UAT sessions within agreed timeline
- All training sessions conducted virtually and recorded for future onboarding
- Admin guide and user quick reference card delivered
- 2-week hypercare monitoring post go-live

#### Phase 2 (V2): Production Platform -- Weeks 13-18

Phase 2 commences upon successful completion of Phase 1 go-live.

**Workstream 10: Production API Setup and Deployment**

Dedicated cloud-hosted API (Railway platform) with automated deployment, environment management, health monitoring, and error logging. Version-controlled codebase that any developer can maintain.

Assumptions:
- Railway cloud platform (~$5-20/month ongoing hosting cost, borne by Client)
- Production code delivered as a version-controlled repository

**Workstream 11: Weight Engine Migration and Test Coverage**

All five product-line modules and shared utilities migrated to the production API with comprehensive test coverage. Each module validated cell-by-cell against its original Excel file. Every calculation path verified against original baselines. No more HubSpot execution time constraints.
